Getting Started

Learn the basics of using SteganoCrypt to hide and reveal secret messages in images.

1

Choose Your Mode

Select either "Encode Message" to hide a secret message in an image, or "Decode Message" to reveal a hidden message.

2

Upload Your Image

Drag and drop your image file or click to browse. Supports PNG, JPG, and JPEG formats up to 10MB.

3

Process

For encoding: enter your secret message. For decoding: the hidden message will be automatically extracted.

4

Download or Copy

Download your encoded image or copy the revealed message to your clipboard for secure sharing.

API Reference

Technical documentation for developers who want to integrate SteganoCrypt functionality.

Note: All processing happens client-side. No data is sent to external servers during encoding/decoding operations.

Encoding Messages

POST /api/encode Content-Type: multipart/form-data Parameters: - image: File (PNG, JPG, JPEG) - message: String (text to hide) Response: - Success: Image file (PNG format) - Error: JSON with error message

Decoding Messages

POST /api/decode Content-Type: multipart/form-data Parameters: - image: File (encoded image) Response: - Success: JSON { "message": "decoded text" } - Error: JSON with error message

Technical Implementation

LSB Steganography Algorithm

SteganoCrypt uses the Least Significant Bit (LSB) steganography technique to embed messages into image pixels.

Binary Conversion

Your message is converted to binary format, with each character represented as 8 bits.

Pixel Modification

The least significant bit of each pixel is modified to store one bit of your message.

Invisible Changes

Changes are so subtle (0-1 value difference) that they're invisible to the human eye.

End Delimiter

A special binary sequence marks the end of your message for accurate extraction.

Security Best Practices

Important: While steganography hides the existence of your message, additional encryption is recommended for sensitive data.
  • Use high-resolution images for better hiding capacity
  • Choose images with complex patterns and textures
  • Keep messages concise to minimize detection risk
  • Always use PNG format for encoded images when possible
  • Share images through secure channels
  • Don't reuse the same cover image multiple times
  • Consider additional encryption for highly sensitive data

Troubleshooting

Upload Issues

Problem: Image won't upload
Solution: Check file size (max 10MB) and format (PNG, JPG, JPEG only)

No Message Found

Problem: "No hidden message found" error
Solution: Ensure the image was encoded with SteganoCrypt and wasn't compressed

Image Quality

Problem: Encoded image looks different
Solution: This shouldn't happen with PNG. Try using a different image format

Mobile Issues

Problem: App not working on mobile
Solution: Ensure you're using a modern browser with JavaScript enabled

Frequently Asked Questions

Q: Is my data sent to your servers?

A: No. All processing happens in your browser. Your images and messages never leave your device.

Q: What's the maximum message length?

A: It depends on image size. As a rule of thumb, you can hide about 1 character per 8 pixels.

Q: Can compressed images be decoded?

A: It depends on compression. PNG (lossless) works perfectly. JPEG may work but isn't guaranteed.

Q: Is this truly secure?

A: Steganography hides message existence, but for maximum security, encrypt your message first.
